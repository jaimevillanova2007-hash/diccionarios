Prueba de Desempeño – Módulo 1 Python
Título:
Sistema de Gestión de Estudiantes con Persistencia de Datos
Caso de uso (Épica):
Como asistente administrativo de una institución educativa, necesitas desarrollar un sistema
sencillo que permita gestionar la información básica de los estudiantes. El sistema debe
permitir registrar, consultar, actualizar y eliminar estudiantes, así como almacenar la
información de manera estructurada para su posterior uso.
Requisitos Funcionales
1. Gestión de estudiantes
El sistema debe permitir:
• Registrar nuevos estudiantes.
• Consultar la lista de estudiantes.
• Buscar un estudiante por algún criterio (ej. ID o nombre).
• Actualizar la información de un estudiante.
• Eliminar estudiantes.
Cada estudiante debe contener como mínimo los siguientes datos:
• ID (identificador único)
• Nombre
• Edad
• Curso o programa
• Estado (activo/inactivo)
2. Almacenamiento en estructuras de datos
La información debe almacenarse utilizando estructuras de datos en memoria, tales
como:
• Listas
• Diccionarios
• Tuplas
*Se permite usar una combinación de estas estructuras si es necesario
